Beta

TIMEFLOW jest w becie. Szukamy freelancerów, którzy chcą kształtować produkt — twój feedback trafia bezpośrednio na roadmapę.

Offline

Działa w pełni bez internetu. Dane lokalne, eksport JSON i transfer przez USB — żadnych wymagań co do połączenia.

Open source

Kod zostanie udostępniony. W przygotowaniu: menedżer projektów z własnym drzewem folderów i archiwizacją plików.

Stack technologiczny

Natywna aplikacja, nie web-app w okienku.

Minimalne wymagania i lekka praca to projekt, a nie przypadek. Backend w Rust, oddzielny daemon i lokalna SQLite dają szybki start i pełną kontrolę nad plikami — bez ciężkiego runtima w tle.

Core Desktop

Rust + Tauri 2

Lekki runtime desktopowy z szybkim startem i natywną kontrolą nad plikami oraz procesami — bez ciężkiego narzutu typowego dla części frameworków desktopowych.

  • lepsza wydajność pod monitoring i pracę w tle
  • natywna integracja z systemem, plikami i procesami
  • mocna baza pod Windows teraz i kolejne platformy desktop

Data Layer

SQLite lokalnie (rusqlite, bundled)

Lokalna baza SQLite działa od razu po instalacji (tryb bundled), więc użytkownik dostaje szybki workflow bez dodatkowego setupu.

  • local-first: dane i dashboard dostępne od ręki
  • szybki odczyt sesji, projektów i analityki
  • backup oraz import/export danych bez kombinowania

UI Dashboard

React 19 + TypeScript + Vite

Dashboard w becie można rozwijać sprawnie — nowe widoki, analityka i panele iterowane niezależnie od backendu.

  • szybka iteracja funkcji i ekranów dashboardu
  • czytelne wykresy i analityka (Recharts)
  • stabilny UI stack (Zustand / Radix / Tailwind)

Daemon

Monitoring w osobnym procesie

Dedykowany daemon w Rust obsługuje pipeline danych niezależnie od UI — dashboard pozostaje interfejsem pracy, a zbieranie aktywności działa w tle.

niski narzut praca w tle oddzielony od UI
  • kontrola start / stop / restart z UI
  • logi i status procesu
  • archiwizacja plików i auto-import workflow
  • autostart i monitoring w tle przy niskim narzucie

Stability Engine

Stabilne dane, czas i zadania w tle

Poprawne dane, precyzyjne rozliczanie czasu i przewidywalne zadania w tle — trzy biblioteki odpowiadają za to, żeby liczby się zgadzały.

  • Serde / serde_json: stabilny import/export i kontrakty danych
  • Chrono: poprawne zakresy dat, sesje i rozliczenia czasu
  • Tokio: timery, zadania cykliczne i operacje w tle

Sync / Web

Serwer synchronizacji — już działa

Serwer na Next.js synchronizuje snapshoty między urządzeniami i zachowuje podejście local-first jako podstawę.

  • synchronizacja snapshotów między urządzeniami
  • API push / pull / ack / status
  • fundament pod mobile i funkcje współdzielenia

Co TIMEFLOW robi już teraz

Realne funkcje, nie lista życzeń.

Wszystko opisane poniżej faktycznie działa w aplikacji — od auto-importu i sesji po sugestie AI, sync online i pełną kontrolę demona.

Tracking Core

Monitoring czasu + auto-import z demona

Daemon zbiera dane w tle, dashboard importuje je przy starcie i odświeża wykryte sesje — przetworzone pliki archiwizowane automatycznie.

  • auto refresh + wykrywanie zmian plików
  • auto-import przy starcie
  • status importu i liczba plików zarchiwizowanych

Dashboard

Widok dnia, tygodnia, miesiąca i all-time

Dashboard pokazuje metryki, top projekty, top aplikacje, timeline projektów oraz ostrzeżenia o nieprzypisanych sesjach.

  • metryki: total, avg daily, apps, projects
  • project timeline + project day timeline
  • manualne sesje z poziomu osi czasu

Projects

Projekty oparte o foldery i auto-detekcję

Projekty z przypisanym folderem, automatyczne wykrywanie kandydatów z aktywności i sync subfolderów — bez ręcznej konfiguracji przy każdym zleceniu.

  • Project Folders (roots) + browse folder
  • Folder Project Candidates + szybkie Create
  • Detected Projects (np. pliki otwierane wielokrotnie)

Sessions

Ręczne porządkowanie sesji bez chaosu

Sesje pogrupowane po projektach, zakres dzienny lub tygodniowy — prawym klikiem przypisujesz, odpinasz lub usuwasz wpis bez wychodzenia z widoku.

  • filtry projektu + fokus z dashboardu
  • prawy klik do przypisywania / odpinania
  • szczegóły plików dla sesji aplikacji

AI & Model

AI sugestie przypisań + tryb auto_safe

Lokalny model uczy się na twoich korektach — bez wysyłania danych na zewnątrz i bez obciążania sprzętu. Tryb suggest albo auto_safe, konfigurowalne progi pewności i rollback ostatniej paczki.

  • training status + reminder po korektach
  • accept/reject sugestii w sesjach
  • rollback last auto-safe batch

Analysis

Analityka czasu: heatmapa, wykresy i eksport CSV

Rozkład czasu po dniach i godzinach, udział aplikacji w pracy oraz eksport CSV — wszystko bez dodatkowej konfiguracji.

  • heatmapa dzień × godzina
  • daily/weekly view
  • time distribution + CSV export

Estimates

Estymacje wartości pracy dla freelancerów

Ustaw globalną stawkę, nadpisz ją per projekt lub dodaj mnożnik do konkretnej sesji — wartość pracy za wybrany zakres widzisz od razu.

  • global hourly rate + project overrides
  • mnożnik stawki dla wybranej sesji
  • estimated value per project
  • liczba projektów i nadpisań stawek

Data & Sync

Import/export danych i online sync

Eksport JSON całości lub projektu, import z walidacją i podglądem konfliktów, snapshot sync online lub przenoszenie przez USB — dane zawsze pod kontrolą.

  • validate import + summary po imporcie
  • backup/export all lub single project
  • startup sync + interval sync + status/ACK

Daemon & Ops

Kontrola demona, logi i autostart

Uruchamiasz, zatrzymujesz i restartujesz daemon z poziomu aplikacji, sprawdzasz logi i monitorujesz stan zbierania danych. Zaprojektowany z myślą o minimalnym śladzie systemowym — nie spowalnia sprzętu w codziennej pracy.

  • start / stop / restart daemon
  • logi z auto-refresh
  • autostart + status + PID
  • niskie wymagania do samego monitoringu

Jak to działa

Od surowych sesji do gotowego rozliczenia.

TIMEFLOW łączy automatyczny tracking z ręczną kontrolą — dokładność bez klikania przez cały dzień.

01

Zbieranie danych

Daemon zbiera dane w tle, dashboard importuje je przy starcie i odświeża wykryte sesje dzienne.

  • daemon status + autostart
  • auto-import przy starcie
  • archiwum plików z importu
02

Porządkowanie i przypisania

Tworzysz projekty, przypisujesz aplikacje, przyjmujesz lub odrzucasz sugestie AI — ręczne korekty tylko tam, gdzie trzeba.

  • projekty + foldery + kandydaci
  • AI suggest / auto_safe / rollback
  • merge sesji według ustawionego gapu
03

Analiza, wycena i backup

Na koniec masz dashboard z metrykami, wycenę czasu pracy i narzędzia do eksportu, importu i synchronizacji między urządzeniami.

  • heatmapa + wykresy
  • estimated value per project + mnożnik sesji
  • JSON export/import + online sync

FAQ

Najczęstsze pytania przed zgłoszeniem do bety.

Krótkie odpowiedzi na najważniejsze kwestie: platformy, dane, koszt i przebieg testów.

Czy TIMEFLOW działa offline?

Tak. Dane są przechowywane lokalnie na urządzeniu, a synchronizacja online jest opcjonalna.

Kto jest priorytetem w testach beta?

Głównie freelancerzy i małe studia rozliczające czas z klientami, ale formularz jest otwarty także dla innych branż.

Na jakiej platformie działa obecna beta?

Aktualnie priorytetem jest Windows. W roadmapie są macOS, Linux i aplikacja mobilna.

Czy beta jest płatna?

Nie. Obecne testy beta są bezpłatne i służą zbieraniu feedbacku do kolejnych iteracji.
