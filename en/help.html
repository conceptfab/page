About the software

TIMEFLOW is an advanced time-tracking ecosystem that works discreetly in the background, letting you focus on what really matters. Unlike traditional tools, TIMEFLOW intelligently analyzes window activity, processes, and files to precisely assign your time to the right projects.

Automatic tracking

The TIMEFLOW Daemon monitors used applications and active documents without your intervention.

Intelligent categorization

A local machine learning engine learns your habits without sending data to the cloud.

Financial analysis

Get immediate insight into the real value of your work with rates and valuation tools.

Privacy & locality

Your data is yours. Everything is stored locally in a secure SQLite database.

AI & MODEL

Proprietary local Rust ML engine analyzing app context, time context and file/window tokens. 100% offline.

TIMEFLOW uses a local classification model. It does not rely on external AI APIs and does not require internet access.

1. Learning signals

  • Application context: app-to-project assignment patterns.
  • Time context: hour/day behavior signatures.
  • Token analysis: keywords from file and window names.

2. Decision metrics

  • Confidence: sigmoid-based probability from 0 to 1.
  • Evidence Count: approved historical matches supporting prediction.
  • Margin: gap between best and second-best candidate.

3. Operating modes

  • Suggest: proposes assignments for user approval.
  • Auto-Safe: automates only high-certainty cases.

4. Recommended settings

  • Mode: suggest for fastest supervised learning.
  • Suggest Min Confidence: 0.4–0.5 to increase learning opportunities.
  • Feedback Weight: 10–15 for fast but stable adaptation.
  • Auto-safe Confidence: 0.85–0.95, Min Evidence: 5 for safe automation.

All model state is stored locally in SQLite and improves with each correction.

Key functionalities

  • Confidence policy threshold management.
  • Learning Center powered by user corrections.
  • Training readiness notifications.
  • Modes: Off, Suggest, Auto-Safe.

Quick Start

Follow these steps to configure TIMEFLOW correctly and start collecting high-quality activity data.

Step 1

File preparation

Place timeflow-dashboard.exe and timeflow-demon.exe in one folder and run the dashboard executable.

Step 2

Projects setup

In Projects, select your parent work directory. Each subfolder becomes a project candidate.

Step 3

Add applications

In Applications, add process names you want to track and assign readable aliases.

Step 4

Start Daemon

In Daemon, start the service and enable autostart so tracking runs in the background.

Step 5

Assign sessions

Use context actions in Dashboard to map unassigned sessions to projects.

Step 6

Train AI

Manual assignment feedback in AI & Model trains the local engine and enables automation.

Done. You are ready to work with a fully configured setup.
