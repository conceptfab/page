Tech stack

A native app. Not a web app in a window.

Minimal footprint by design, not by accident. Rust backend, a separate daemon and local SQLite give you fast startup and full file control — no heavy runtime running in the background.

Core Desktop

Rust + Tauri 2

Lightweight desktop runtime with fast startup and native control over files and processes — without the overhead common to some desktop frameworks.

  • better performance for background monitoring and work
  • native integration with the OS, files and processes
  • solid foundation for Windows now and other desktop platforms later

Data Layer

SQLite locally (rusqlite, bundled)

Local SQLite works right out of the box (bundled mode), so you get a fast workflow without any extra setup.

  • local-first: data and dashboard ready immediately
  • fast reads for sessions, projects and analytics
  • backup and import/export without the hassle

UI Dashboard

React 19 + TypeScript + Vite

The beta dashboard can be iterated quickly — new views, analytics and panels developed independently from the backend.

  • fast iteration of features and dashboard screens
  • clean charts and analytics (Recharts)
  • stable UI stack (Zustand / Radix / Tailwind)

Daemon

Monitoring in a separate process

A dedicated Rust daemon handles the data pipeline independently from the UI — the dashboard stays your work interface while activity collection runs in the background.

low overhead runs in background separate from UI
  • start / stop / restart from UI
  • process logs and status
  • file archiving and auto-import workflow
  • autostart and background monitoring with low overhead

Stability Engine

Stable data, time and background tasks

Accurate data, precise time calculations and predictable background tasks — three core libraries that make sure the numbers add up.

  • Serde / serde_json: stable import/export and data contracts
  • Chrono: correct date ranges, sessions and time calculations
  • Tokio: timers, recurring tasks and background operations

Sync / Web

Sync server — already running

Next.js syncs snapshots between devices while keeping local-first as the foundation.

  • snapshot sync between devices
  • API push / pull / ack / status
  • foundation for mobile and sharing features

Projects linked to folders, automatic candidate detection from activity and subfolder sync — no manual setup for every new project.

Sessions

Sessions grouped by project, daily or weekly range — right-click to assign, unpin or remove an entry without leaving the view.

A local model learns from your corrections — no data sent out, no hardware load. suggest or auto_safe mode, configurable confidence thresholds and rollback of the last batch.

How it works

From raw sessions to a finished invoice.

TIMEFLOW combines automatic tracking with manual control — accuracy without clicking all day.

01

Collecting data

The daemon collects data in the background, the dashboard imports it on startup and refreshes detected daily sessions.

  • daemon status + autostart
